# ChartForge

ChartForge is an R package and point-and-click web application for designing JavaScript charts from R data frames and matrices. The app emits reusable R code, evaluates that code through local R or WebR, and previews the HTML/CSS/JavaScript string returned by the selected ChartForge R function.

You can use the webR version at https://alekrutkowski.github.io/ChartForge/.

The emitted R code is not a pasted HTML literal. It can be rerun outside the app with new or modified data:

```r
library(chartforge)

df <- demo_data()

chart_html <- cf_plotly_scatter(
  data = df,
  x = "wt",
  y = "mpg",
  group = "cyl",
  label = c("model", "am"),
  sort_by = "wt",
  title = "Fuel economy by vehicle weight",
  subtitle = "Generated by ChartForge",
  width = 840,
  height = 480,
  theme = "aurora",
  palette = cf_palette("studio"),
  legend = "show",
  legend_position = "bottom",
  font_family = "Inter, sans-serif",
  decimal_mark = ".",
  renderer_style = list(
    scroll_zoom = TRUE,
    hovermode = "closest",
    marker_opacity = 0.9
  ),
  full_bundle = FALSE,
  embed = TRUE,
  minified = FALSE
)

cf_save_html(chart_html, "chart.html")
```

The MIT licence covers ChartForge code only. It does not grant rights to use third-party renderers.

## App workflow

The compact two-pane interface keeps settings on the left and output on the right. Five settings tabs cover:

1. **Data**: paste or upload CSV, TSV, or semicolon-delimited data; control the input decimal mark and ISO date/date-time recognition; or import a data frame or matrix from the R session that launched the app.
2. **Chart**: choose a JavaScript renderer and chart type, then edit titles and chart-type parameters.
3. **Mappings**: map x, y, group/colour, multiple hover-label columns, and sorting.
4. **Style**: use controls generated from a renderer-specific schema. The controls change with the selected library and expose its useful interactions, marks, scales, axes, navigation, animation, and performance options. Official documentation and licence links are shown for the current renderer.
5. **General settings**: set exact pixel dimensions, theme, palette, Google Font, typography, DPI, decimal mark, legend, grid, tooltip, animation, fragment/page output, vendor bundling, and minification.

The right pane provides live chart preview, syntax-highlighted R code, syntax-highlighted HTML output, and a data preview. Both code panels follow the app's light/dark setting. Local-R and WebR errors are displayed as readable error cards in the chart area.

## Install and run

```r
install.packages(c("plumber", "jsonlite", "remotes"))
remotes::install_github("alekrutkowski/ChartForge")
chartforge::run_app()
```

Data frames and matrices visible from the calling environment are exposed automatically:

```r
analysis_data <- transform(mtcars, model = row.names(mtcars))
chartforge::run_app()
```

A specific environment can be supplied explicitly:

```r
chartforge::run_app(data_env = my_environment)
```

From a source checkout:

```bash
Rscript scripts/install-deps.R
Rscript scripts/run-local.R
```

## Reusable R API

ChartForge exports one wrapper for every supported renderer and chart-type combination. The current chart types are:

`scatter`, `line`, `spline`, `step`, `bar`, `stacked_bar`, `area`, `stacked_area`, `histogram`, `density`, `pie`, `donut`, `bubble`, `boxplot`, `heatmap`, `treemap`, and `lollipop`.

The renderers are:

- Highcharts
- Plotly.js
- D3.js
- Vega-Lite / Vega
- Observable Plot
- Chart.js
- Apache ECharts
- ApexCharts
- Google Charts
- amCharts 5
- VisActor VChart
- AnyChart
- Billboard.js

Examples:

```r
cf_highcharts_bar(df, x = "category", y = "value")
cf_echarts_line(df, x = "date", y = "value", group = "series")
cf_vega_histogram(df, x = "value")
cf_plotly_boxplot(df, x = "group", y = "value")
cf_google_scatter(df, x = "wt", y = "mpg")
```

Use `cf_supported()` for the complete wrapper grid, `chart_libraries()` for renderer versions, licence policies, and documentation links, and `cf_renderer_styles()` for the renderer-specific style schemas used by the app.

### Data import, sorting, and hover labels

```r
df <- cf_from_delim(
  "date\tseries\tvalue\n2026-01-01\tA\t1.5\n2026-01-02\tB\t2.5",
  delim = NULL,
  auto_dates = TRUE,
  decimal_mark = "."
)

cf_vchart_scatter(
  df,
  x = "date",
  y = "value",
  group = "series",
  label = c("series", "date"),
  sort_by = c("series", "date"),
  sort_desc = c(FALSE, FALSE)
)
```

Mapped x, y, and group columns are automatically removed from extra hover-label text, and repeated label values are deduplicated.

### Renderer-specific styles

The `renderer_style` argument accepts the friendly options shown in the Style tab:

```r
cf_highcharts_scatter(
  df,
  x = "wt",
  y = "mpg",
  renderer_style = list(
    zoom_type = "xy",
    crosshair = TRUE,
    exporting = TRUE
  )
)

cf_echarts_line(
  df,
  x = "date",
  y = "value",
  group = "series",
  renderer_style = list(
    data_zoom = TRUE,
    toolbox = TRUE,
    smooth = TRUE,
    axis_pointer = "cross"
  )
)
```

Advanced native option lists remain available, including `highcharts_options`, `plotly_layout`, `plotly_config`, `vega_config`, `chartjs_options`, `echarts_options`, `apexcharts_options`, `google_options`, `amcharts_options`, `vchart_options`, `anychart_options`, and `billboard_options`.

## Output modes and licence-aware bundling

The defaults are:

```r
full_bundle = FALSE
embed = TRUE
minified = FALSE
```

With `full_bundle = FALSE`, each vendor library is referenced using a compact CDN tag. ChartForge adds only the runtime specialized to the selected renderer and chart type.

With `full_bundle = TRUE`:

| Renderer policy | Behavior |
|---|---|
| Permissive | Vendor JavaScript/CSS is inlined and the exact licence/notice text is embedded in the HTML. |
| Conditional | Vendor assets are inlined only with the applicable conditions and notices preserved. This currently applies to amCharts 5. |
| CDN only | Vendor assets remain remotely linked. The HTML records that ChartForge deliberately did not download or redistribute them. |

The CDN-only renderers are Highcharts, ApexCharts, Google Charts, and AnyChart. CDN delivery reduces redistribution risk, but it does not remove the end user's obligation to comply with the renderer's terms.

Set `minified = TRUE` for compact HTML/CSS/JavaScript. Set `embed = FALSE` for a complete HTML document. `asset_paths` can provide local files only for renderers whose bundle policy permits inlining.

A generated HTML fragment begins with a notice similar to:

```html
<!--
ChartForge third-party renderer notice
Renderer: ApexCharts 5.16.0
Licence or terms: ApexCharts Community or commercial licence (https://apexcharts.com/license/community/)
Delivery: full_bundle=TRUE was requested, but ChartForge intentionally kept this vendor library CDN-linked and did not download or inline it.
Important: eligibility and permitted use depend on the current ApexCharts licence terms.
ChartForge's MIT licence does not grant rights to this renderer.
-->
```

## Package size and release layout

ChartForge does not ship third-party vendor JavaScript distributions. The application contains ChartForge's own renderer adapters plus exact text licence notices.

Renderer-specific adapter code is stored in two gzip-compressed JSON packs:

- `runtime-readable.json.gz`
- `runtime-minified.json.gz`

Each pack is indexed by renderer and chart type. At runtime, only the selected entry is inserted into generated HTML. The static GitHub Pages site is deployed from `inst/app`, so the R package and website no longer contain duplicate copies of the same application and runtime files.

## GitHub publication audit

Before committing a release, run:

```bash
npm ci --ignore-scripts
npm run audit:release
npm test
```

The audit parses every JSON file, rejects duplicate JSON keys, verifies both compressed runtime packs and all shipped third-party notice hashes, rejects private registry URLs and local build paths, checks common secret signatures and cross-platform file names, confirms the public npm registry lockfile, validates workflow references, and ensures the repository contains `LICENSE.md` but no duplicate plain `LICENSE` file. `node_modules/` is never committed or packaged.
